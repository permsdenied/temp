# Telegram-бот для генерации коммерческих предложений

Бот принимает запрос на составление КП, генерирует контент через Google Gemini API, подставляет его в `.docx` шаблон и отправляет готовый файл в Telegram. Пользователь может запросить правки — бот учитывает историю диалога и присылает обновлённую версию.

---

## Структура проекта

```
├── bot.py              # Основной файл бота
├── gemini_client.py    # Клиент Google Gemini API
├── docx_handler.py     # Заполнение .docx шаблона
├── requirements.txt
├── .env.example
└── КП_№SP26-10_от_18_02_26г (1) (1) (1).docx   # Шаблон
```

---

## 1. Установка зависимостей

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

---

## 2. Получение Telegram Bot Token

1. Открой Telegram и найди **@BotFather**.
2. Отправь команду `/newbot` и следуй инструкциям.
3. Скопируй выданный токен вида `7123456789:AAF...`.

---

## 3. Получение Gemini API Key

1. Перейди на [https://aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey).
2. Нажми **Create API key**.
3. Скопируй ключ.

---

## 4. Настройка `.env`

Скопируй `.env.example` в `.env` и заполни значения:

```bash
cp .env.example .env
```

```env
TELEGRAM_BOT_TOKEN=7123456789:AAF...
GEMINI_API_KEY=AIza...
TEMPLATE_PATH=КП_№SP26-10_от_18_02_26г (1) (1) (1).docx
```

---

## 5. Добавление плейсхолдеров в `.docx` шаблон

Откройте шаблон в Microsoft Word / LibreOffice Writer и замените нужные текстовые блоки на плейсхолдеры. **Логотипы, адреса и телефоны в колонтитулах трогать не нужно.**

| Плейсхолдер | Что подставится |
|---|---|
| `{{kp_number}}` | Номер КП, например SP26-01 |
| `{{kp_date}}` | Дата КП, например 29.03.2026 |
| `{{company_name}}` | Название компании клиента |
| `{{contact_person}}` | Контактное лицо |
| `{{service_title}}` | Название услуги / продукта |
| `{{service_description}}` | Описание услуги |
| `{{intro_text}}` | Вводный текст КП |
| `{{outro_text}}` | Заключительный текст / призыв к действию |
| `{{total_amount}}` | Итоговая сумма |
| `{{validity_period}}` | Срок действия КП |
| `{{price_table}}` | Маркер таблицы цен (см. ниже) |

### Таблица цен

Создайте в шаблоне таблицу со следующей структурой:

| № | Наименование | Кол-во | Ед. | Цена | Итого |
|---|---|---|---|---|---|
| (заголовок) | | | | | |
| `{{price_table}}` | | | | | |
| (строка итогов, опционально) | | | | | |

Строка с `{{price_table}}` будет использована как шаблон для каждой позиции прайс-листа.

---

## 6. Запуск бота

```bash
python bot.py
```

Напишите боту команду `/start` и отправьте запрос, например:

> Составь КП для компании «Альфа Девелопмент», услуга: разработка корпоративного сайта, сумма: 800 000 тенге

Бот сгенерирует документ и отправит `.docx` файл. Для правок — просто напишите что изменить:

> Поменяй сумму на 900 000 тенге и сократи введение

---

## 7. Деплой

### Railway

1. Создай аккаунт на [railway.app](https://railway.app).
2. Нажми **New Project → Deploy from GitHub repo** и выбери этот репозиторий.
3. В разделе **Variables** добавь переменные из `.env`.
4. Railway автоматически установит зависимости и запустит `bot.py`.

### Render

1. Создай аккаунт на [render.com](https://render.com).
2. Нажми **New → Web Service**, выбери репозиторий.
3. Укажи:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python bot.py`
4. Добавь переменные окружения в разделе **Environment**.

---

## Примечания

- Сгенерированные `.docx` файлы временно сохраняются в папку `temp_output/` и удаляются после отправки.
- История диалога хранится в памяти процесса. При перезапуске бота история сбрасывается.
- Если Gemini вернул некорректный JSON, бот автоматически повторяет запрос до 3 раз.
